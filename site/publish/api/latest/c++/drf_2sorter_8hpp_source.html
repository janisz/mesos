45  // We store the number of times this client has been chosen for
46  // allocation so that we can fairly share the resources across
47  // clients that have the same share. Note that this information is
48  // not persisted across master failovers, but since the point is to
49  // equalize the 'allocations' across clients of the same 'share'
50  // having allocations restart at 0 after a master failover should be
51  // sufficient (famous last words.)
