41 // The scheme an external containerizer programs have to adhere to is;
42 //
43 // COMMAND < INPUT-PROTO > RESULT-PROTO
44 //
45 // launch < containerizer::Launch
46 // update < containerizer::Update
47 // usage < containerizer::Usage > mesos::ResourceStatistics
48 // wait < containerizer::Wait > containerizer::Termination
49 // destroy < containerizer::Destroy
50 // containers > containerizer::Containers
51 // recover
52 //
53 // 'wait' on the external containerizer side is expected to block
54 // until the task command/executor has terminated.
55 //
56 // Additionally, we have the following environment variable setup
57 // for external containerizer programs:
58 // MESOS_LIBEXEC_DIRECTORY = path to mesos-executor, mesos-usage, ...
59 // MESOS_WORK_DIRECTORY = slave work directory. This should be used
60 // for distiguishing slave instances.
61 // MESOS_DEFAULT_CONTAINER_IMAGE = default image as provided via
62 // slave flags (default_container_image). This variable is provided
63 // only in calls to 'launch'.
64 
65 // Check src/examples/python/test_containerizer.py for a rough
66 // implementation template of this protocol.
67 
68 // For debugging purposes of an external containerizer program, it
69 // might be helpful to enable verbose logging on the slave (GLOG_v=2).
