89  // This environment decorator is called from within the slave after
90  // receiving a run task request from the master but before the docker
91  // containerizer launches the task. A module implementing the hook can
92  // inspect the arguments and return a `Failure` if the task should be
93  // rejected with a `TASK_FAILED`.
94  // The hook can return a set of environment variables. For command tasks
95  // the environment variables will become part of the task's environment.
96  // For custom executors, the environment variables will be part of the
97  // custom executor's environment.
98  //
99  // NOTE: The order of hooks matters for environment variables.
100  // If there is a conflict, the hook loaded last will take priority.
101  //
102  // NOTE: Unlike `slaveExecutorEnvironmentDecorator`, environment variables
103  // returned from this hook *will* overwrite environment variables inside
104  // the `ExecutorInfo`.
105  //
106  // NOTE: This hook is designed to be an asynchronous replacement for
107  // `slavePreLaunchDockerHook`. This hook is called first.
