65 /*
66  * TODO(benh): Clean up this documentation.
67  *
68  * This is the main class of ZooKeeper client library. To use a
69  * ZooKeeper service, an application must first instantiate an object
70  * of ZooKeeper class. All the iterations will be done by calling the
71  * methods of ZooKeeper class.
72  *
73  * Once a connection to a server is established, a session ID is
74  * assigned to the client. The client will send heart beats to the
75  * server periodically to keep the session valid.
76  *
77  * The application can call ZooKeeper APIs through a client as long as
78  * the session ID of the client remains valid.
79  *
80  * If for some reason, the client fails to send heart beats to the
81  * server for a prolonged period of time (exceeding the sessionTimeout
82  * value, for instance), the server will expire the session, and the
83  * session ID will become invalid. The client object will no longer be
84  * usable. To make ZooKeeper API calls, the application must create a
85  * new client object.
86  *
87  * If the ZooKeeper server the client currently connects to fails or
88  * otherwise does not respond, the client will automatically try to
89  * connect to another server before its session ID expires. If
90  * successful, the application can continue to use the client.
91  *
92  * Some successful ZooKeeper API calls can leave watches on the "data
93  * nodes" in the ZooKeeper server. Other successful ZooKeeper API
94  * calls can trigger those watches. Once a watch is triggered, an
95  * event will be delivered to the client which left the watch at the
96  * first place. Each watch can be triggered only once. Thus, up to one
97  * event will be delivered to a client for every watch it leaves.
98  *
99  * A client needs an object of a class implementing Watcher interface
100  * for processing the events delivered to the client. When a client
101  * drops current connection and re-connects to a server, all the
102  * existing watches are considered as being triggered but the
103  * undelivered events are lost. To emulate this, the client will
104  * generate a special event to tell the event handler a connection has
105  * been dropped. This special event has type EventNone and state
106  * sKeeperStateDisconnected.
107  */
