152  // This is a weak pointer to 'this', i.e., ourselves, this class
153  // instance. We need this for our event loop callbacks because it's
154  // possible that we'll actually want to cleanup this socket impl
155  // before the event loop callback gets executed ... and we'll check
156  // in each event loop callback whether or not this weak_ptr is valid
157  // by attempting to upgrade it to shared_ptr. It is the
158  // responsibility of the event loop through the deferred lambda in
159  // the destructor to clean up this pointer.
160  // 1) It is a 'weak_ptr' as opposed to a 'shared_ptr' because we
161  // want to test whether the object is still around from within the
162  // event loop. If it was a 'shared_ptr' then we would be
163  // contributing to the lifetime of the object and would no longer be
164  // able to test the lifetime correctly.
165  // 2) This is a pointer to a 'weak_ptr' so that we can pass this
166  // through to the event loop through the C-interface. We need access
167  // to the 'weak_ptr' from outside the object (in the event loop) to
168  // test if the object is still alive. By maintaining this 'weak_ptr'
169  // on the heap we can be sure it is safe to access from the
170  // event loop until it is destroyed.
171  std::weak_ptr<LibeventSSLSocketImpl>* event_loop_handle;
