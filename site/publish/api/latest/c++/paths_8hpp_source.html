33 // The slave leverages the file system for a number of purposes:
34 //
35 // (1) For executor sandboxes and disk volumes.
36 //
37 // (2) For checkpointing metadata in order to support "slave
38 // recovery". That is, allow the slave to restart without
39 // affecting the tasks / executors.
40 //
41 // (3) To detect reboots (by checkpointing the boot id), in which
42 // case everything has died so no recovery should be attempted.
43 //
44 // (4) For checkpointing resources that persist across slaves.
45 // This includes things like persistent volumes and dynamic
46 // reservations.
47 //
48 // (5) For provisioning root filesystems for containers.
49 //
50 // The file system layout is as follows:
51 //
52 // root ('--work_dir' flag)
53 // |-- slaves
54 // | |-- latest (symlink)
55 // | |-- <slave_id>
56 // | |-- frameworks
57 // | |-- <framework_id>
58 // | |-- executors
59 // | |-- <executor_id>
60 // | |-- runs
61 // | |-- latest (symlink)
62 // | |-- <container_id> (sandbox)
63 // |-- meta
64 // | |-- slaves
65 // | |-- latest (symlink)
66 // | |-- <slave_id>
67 // | |-- slave.info
68 // | |-- frameworks
69 // | |-- <framework_id>
70 // | |-- framework.info
71 // | |-- framework.pid
72 // | |-- executors
73 // | |-- <executor_id>
74 // | |-- executor.info
75 // | |-- runs
76 // | |-- latest (symlink)
77 // | |-- <container_id> (sandbox)
78 // | |-- executor.sentinel (if completed)
79 // | |-- pids
80 // | | |-- forked.pid
81 // | | |-- libprocess.pid
82 // | |-- tasks
83 // | |-- <task_id>
84 // | |-- task.info
85 // | |-- task.updates
86 // |-- boot_id
87 // |-- resources
88 // | |-- resources.info
89 // |-- volumes
90 // | |-- roles
91 // | |-- <role>
92 // | |-- <persistence_id> (persistent volume)
93 // |-- provisioner
