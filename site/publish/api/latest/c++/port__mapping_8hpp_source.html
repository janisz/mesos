137 // Provides network isolation using port mapping. Each container is
138 // assigned a fixed set of ports (including ephemeral ports). The
139 // isolator will set up filters on the host such that network traffic
140 // to the host will be properly redirected to the corresponding
141 // container depending on the destination ports. The network traffic
142 // from containers will also be properly relayed to the host. This
143 // isolator is useful when the operator wants to reuse the host IP for
144 // all containers running on the host (e.g., there are insufficient
145 // IPs).
