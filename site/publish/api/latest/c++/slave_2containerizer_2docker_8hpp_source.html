407  // The DockerContainerizer needs to be able to properly clean up
408  // Docker containers, regardless of when they are destroyed. For
409  // example, if a container gets destroyed while we are fetching,
410  // we need to not keep running the fetch, nor should we try and
411  // start the Docker container. For this reason, we've split out
412  // the states into:
413  //
414  // FETCHING
415  // PULLING
416  // MOUNTING
417  // RUNNING
418  // DESTROYING
419  //
420  // In particular, we made 'PULLING' be it's own state so that we
421  // can easily destroy and cleanup when a user initiated pulling
422  // a really big image but we timeout due to the executor
423  // registration timeout. Since we currently have no way to discard
424  // a Docker::run, we needed to explicitly do the pull (which is
425  // the part that takes the longest) so that we can also explicitly
426  // kill it when asked. Once the functions at Docker::* get support
427  // for discarding, then we won't need to make pull be it's own
428  // state anymore, although it doesn't hurt since it gives us
429  // better error messages.
430  enum State
431  {
432  FETCHING = 1,
433  PULLING = 2,
434  MOUNTING = 3,
435  RUNNING = 4,
436  DESTROYING = 5
437  } state;
