Detailed Description

A hook can be passed to a subprocess call.

It provides a way to inject dynamic implementation behavior between the clone and exec calls in the implementation of subprocess.

Member Data Documentation

const lambda::function<Try<Nothing>pid_t)> process::Subprocess::Hook::parent_callback

The callback that must be sepcified for execution after the child has been cloned, but before it start executing the new process.

This provides access to the child pid after its initialization to add tracking or modify execution state of the child before it executes the new process.
